Preamble Preâmbulo

Architecture was designed for systems that hold still.

A arquitetura foi concebida para sistemas que ficam parados.

A solution architect would model the problem, define the components, document the interfaces, and publish the design. The system would be built to match it. The architecture would persist as a reference — reviewed periodically, updated occasionally, consulted when something broke.

Um arquiteto de soluções modelaria o problema, definiria os componentes, documentaria as interfaces e publicaria o design. O sistema seria construído para corresponder a ele. A arquitetura persistiria como referência — revisada periodicamente, atualizada ocasionalmente, consultada quando algo quebrasse.

This model assumed that the distance between a system as designed and a system as running could be managed through deliberate, scheduled intervention. It assumed that the architectural intent embedded in a document at sprint zero would remain legible in the system running twelve months later.

Esse modelo pressupunha que a distância entre um sistema como projetado e um sistema em execução poderia ser gerenciada por meio de intervenção deliberada e programada. Pressupunha que a intenção arquitetural incorporada em um documento no sprint zero permaneceria legível no sistema em execução doze meses depois.

Neither assumption holds anymore.

Nenhuma dessas premissas vale mais.

Modern systems evolve in production — continuously, through automated pipelines, composed dependencies, distributed teams, and adversarial environments that do not wait for the next review cycle. The architectural artifact ages from the moment it is published. The distance between designed intent and operational reality grows with every change that is not treated as an architectural event.

Sistemas modernos evoluem em produção — continuamente, por meio de pipelines automatizados, dependências compostas, times distribuídos e ambientes adversariais que não esperam pelo próximo ciclo de revisão. O artefato arquitetural envelhece no momento em que é publicado. A distância entre a intenção do design e a realidade operacional cresce a cada mudança que não é tratada como um evento arquitetural.

This is not a process problem. It is a model problem.

Isso não é um problema de processo. É um problema de modelo.

The traditional model treats architecture as an initial state to be preserved. The model we need treats architecture as a living state to be governed.

O modelo tradicional trata a arquitetura como um estado inicial a ser preservado. O modelo que precisamos trata a arquitetura como um estado vivo a ser governado.

Values Valores

We have come to value:

Passamos a valorizar:

  • Continuous architectural posture Postura arquitetural contínua over em vez de point-in-time design artifacts artefatos de design pontuais
  • Codified judgment Julgamento codificado over em vez de manual review revisão manual
  • Drift detection Detecção de drift over em vez de periodic audit auditoria periódica
  • Adaptive governance Governança adaptativa over em vez de fixed compliance conformidade fixa
  • Composition awareness Consciência de composição over em vez de boundary isolation isolamento de fronteiras

That is, while there is value in the items on the right, we value the items on the left more.

Ou seja, embora haja valor nos itens à direita, valorizamos mais os itens à esquerda.

Principles Princípios

  1. 01

    An architecture that is not continuously observed is not an architecture. It is a historical document.

    Uma arquitetura que não é observada continuamente não é uma arquitetura. É um documento histórico.

  2. 02

    Every deployment is a potential architectural event. Treat it as one.

    Todo deployment é um potencial evento arquitetural. Trate-o como tal.

  3. 03

    The architect's judgment is most valuable when it does not require the architect's presence.

    O julgamento do arquiteto é mais valioso quando não exige a presença do arquiteto.

  4. 04

    The gap between intended architecture and running system is not an implementation problem. It is an architecture problem.

    A distância entre a arquitetura intencionada e o sistema em execução não é um problema de implementação. É um problema de arquitetura.

  5. 05

    Governance that cannot observe the system it governs is not governance. It is ceremony.

    Governança que não consegue observar o sistema que governa não é governança. É cerimônia.

  6. 06

    Requirements generated once and never revisited are already wrong. Systems change; requirements must follow.

    Requisitos gerados uma vez e nunca revisitados já estão errados. Sistemas mudam; requisitos precisam seguir.

  7. 07

    Complexity that cannot be expressed cannot be governed. If you cannot model it, you cannot reason about it.

    Complexidade que não pode ser expressa não pode ser governada. Se você não consegue modelá-la, não consegue raciocinar sobre ela.

  8. 08

    The value of a design decision lives not in the document that records it but in the system that reflects it.

    O valor de uma decisão de design não reside no documento que a registra, mas no sistema que a reflete.

  9. 09

    Threat surfaces, technical debt, and compliance gaps do not appear suddenly. They accumulate — silently, across every change that was not treated as an architectural event.

    Superfícies de ataque, dívida técnica e lacunas de conformidade não aparecem de repente. Eles se acumulam — silenciosamente, a cada mudança que não foi tratada como um evento arquitetural.

  10. 10

    The role of the architect is not to review decisions one at a time. It is to design the system by which decisions govern themselves.

    O papel do arquiteto não é revisar decisões uma a uma. É projetar o sistema pelo qual as decisões se governam.

Architecture was never meant to be a snapshot. It was always meant to be a commitment about how a system should behave — now, and as it changes. We have confused the document with the discipline. This manifesto is a call to recover the discipline.

A arquitetura nunca foi pensada para ser um snapshot. Sempre foi uma afirmação sobre como um sistema deve se comportar — agora e conforme ele muda. Confundimos o documento com a disciplina. Este manifesto é um chamado para recuperar a disciplina.
